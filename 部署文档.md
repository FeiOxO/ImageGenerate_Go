# 部署文档

本文档说明如何将 AI 生图 Go 后端部署到 Linux 服务器。以下示例以 Ubuntu Server 为例。

## 服务器要求

- Linux 服务器一台
- Go 1.22 或更高版本
- MySQL 8.0 或兼容版本
- 可访问外部生图接口的网络环境
- 已配置好 Git SSH Key 或具备代码上传方式

## 1. 创建数据库

登录 MySQL：

```bash
mysql -u root -p
```

创建数据库和业务账号：

```sql
CREATE DATABASE ai_image_demo
  CHARACTER SET utf8mb4
  COLLATE utf8mb4_unicode_ci;

CREATE USER 'ai_image_user'@'%' IDENTIFIED BY '请替换为强密码';
GRANT ALL PRIVILEGES ON ai_image_demo.* TO 'ai_image_user'@'%';
FLUSH PRIVILEGES;
```

生产环境建议限制数据库账号来源 IP，不建议长期使用 `%`。

## 2. 获取代码

```bash
git clone git@github.com:FeiOxO/ImageGenerate_Go.git
cd ImageGenerate_Go
```

## 3. 配置环境变量

复制环境变量模板：

```bash
cp .env.example .env
```

编辑 `.env`：

```env
APP_PORT=3000

JWT_SECRET=请替换为足够长的随机字符串
JWT_EXPIRE_HOURS=168

MYSQL_DSN=ai_image_user:请替换为强密码@tcp(127.0.0.1:3306)/ai_image_demo?charset=utf8mb4&parseTime=True&loc=Local

IMAGE_API_BASE_URL=https://www.codexapis.com
IMAGE_API_APP_ID=your-app-id
IMAGE_API_KEY=your-api-key
IMAGE_OUTPUT_DIR=./storage/images
WORKER_POOL_SIZE=8
```

注意事项：

- `.env` 不能提交到 Git。
- `JWT_SECRET` 必须使用生产环境专用密钥。
- `IMAGE_API_KEY`、`IMAGE_API_APP_ID`、`MYSQL_DSN` 都属于敏感信息。
- 如果图片量较大，建议将 `IMAGE_OUTPUT_DIR` 放到独立数据盘。

## 4. 构建服务

安装依赖并构建：

```bash
go mod tidy
go build -o ai-image-api ./cmd/api
```

测试启动：

```bash
./ai-image-api
```

访问健康检查：

```bash
curl http://127.0.0.1:3000/api/health
```

返回以下内容表示服务正常：

```json
{
  "status": "ok"
}
```

## 5. 使用 systemd 托管服务

创建服务文件：

```bash
sudo nano /etc/systemd/system/ai-image-api.service
```

写入以下内容，并按实际部署路径修改 `WorkingDirectory` 和 `ExecStart`：

```ini
[Unit]
Description=AI Image Generate Go API
After=network.target mysql.service

[Service]
Type=simple
WorkingDirectory=/opt/ImageGenerate_Go
ExecStart=/opt/ImageGenerate_Go/ai-image-api
Restart=always
RestartSec=5
EnvironmentFile=/opt/ImageGenerate_Go/.env

[Install]
WantedBy=multi-user.target
```

重新加载并启动：

```bash
sudo systemctl daemon-reload
sudo systemctl enable ai-image-api
sudo systemctl start ai-image-api
```

查看状态：

```bash
sudo systemctl status ai-image-api
```

查看日志：

```bash
journalctl -u ai-image-api -f
```

## 6. Nginx 反向代理

如果需要通过域名访问，可以使用 Nginx 代理到本地 `3000` 端口。

示例配置：

```nginx
server {
    listen 80;
    server_name api.example.com;

    client_max_body_size 20m;

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

启用配置后检查并重载：

```bash
sudo nginx -t
sudo systemctl reload nginx
```

生产环境建议再配置 HTTPS。

## 7. 图片目录权限

服务会将生成图片保存到 `IMAGE_OUTPUT_DIR`。如果使用 systemd 运行，需要确保运行服务的用户有写入权限。

示例：

```bash
mkdir -p /opt/ImageGenerate_Go/storage/images
chmod -R 755 /opt/ImageGenerate_Go/storage
```

如果使用专用用户运行服务，应将目录所有者改为该用户。

## 8. 常用维护命令

重启服务：

```bash
sudo systemctl restart ai-image-api
```

停止服务：

```bash
sudo systemctl stop ai-image-api
```

查看最近日志：

```bash
journalctl -u ai-image-api -n 100
```

更新代码并重新部署：

```bash
git pull
go build -o ai-image-api ./cmd/api
sudo systemctl restart ai-image-api
```

## 9. 安全建议

- 不要把 `.env` 上传到 Git。
- 生产环境必须替换 `JWT_SECRET`。
- MySQL 账号只授予当前数据库权限。
- 限制 MySQL 端口访问来源。
- 建议通过 HTTPS 暴露 API。
- 建议定期备份 MySQL 数据库和图片目录。
- `WORKER_POOL_SIZE` 不要盲目调大，过高会增加外部 API 压力、内存占用和失败率。
